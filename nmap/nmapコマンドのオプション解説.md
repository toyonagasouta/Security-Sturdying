# nmapコマンドのオプション解説

## -P系:「Ping」(ホスト検出)を意味する

-Pで始まるオプションは、実際にポートスキャンを行う前段階の「ホストが生きているかどうか」を確認する**ホストディスカバリ**のためのものです。ここでの"P"は文字通り"Ping"の頭文字です。

- `-PE`: ICMP Echo Request(伝統的なping)
- `-PS`: TCP SYNパケットによるping(SYN ping)
- `-PA`: TCP ACKパケットによるping(ACK ping)
- `-PU`: UDPパケットによるping(UDP ping)
- `-PR`: ARPによるping(同一LAN内で使用)
- `-PY`: SCTPによるping
- `-PO`: IPプロトコルping

※ `-PT`は古いバージョンのnmapに存在したオプション(TCP ping)で、現在は`-PS`に統合・置き換えられています。man pageやhelpにはもう出てこないはずです。

## -S系:「Scan」(ポートスキャン手法)を意味する

-sで始まるオプションは、トランスポート層(主にTCP/UDP)で実際にポートの開閉状態を調べる**スキャン手法の種類**を指定します。こちらは"scan"の頭文字です。

- `-sS`: TCP SYNスキャン(ステルススキャン、デフォルト)
- `-sT`: TCP Connectスキャン(3ウェイハンドシェイクを完了させる)
- `-sU`: UDPスキャン
- `-sA`: TCP ACKスキャン(ファイアウォールのルール調査用)
- `-sF` / `-sN` / `-sX`: FIN/Null/Xmasスキャン
- `-sO`: IPプロトコルスキャン

## まとめ

| プレフィックス | 意味 | 目的 |
|---|---|---|
| `-P*` | **P**ing | ホストが応答するか(生存確認) |
| `-s*` | **s**can | どのポートが開いているか(スキャン手法) |

大文字部分(P/s)がオプションの「カテゴリ」を表し、それに続くアルファベット1〜2文字が使用するプロトコルや手法の頭文字(E=Echo, S=SYN, U=UDP, T=Connectなど)になっている、という設計です。これはnmapの他のオプション(例えば`-O`=OS detection、`-A`=Aggressive)にも通じる、比較的直感的な命名規則と言えます。
