#　nmapコマンドについて

# nmapコマンドの基本構文とペネトレーションテストで役立つオプション

## 基本構文

```
nmap [スキャンタイプ] [オプション] [ターゲット]
```

例：
```bash
nmap -sS -sV -O -p- 192.168.1.1
```

---

## 1. ホストスキャン（ホスト検出）オプション

対象ホストが生きているかを確認するためのオプションです。

| オプション | 説明 |
|---|---|
| `-sn` | Ping スキャンのみ（ポートスキャンをしない、旧`-sP`） |
| `-Pn` | ホスト検出をスキップし、全ホストをオンラインとみなしてスキャン（ファイアウォールでping拒否されている環境で有効） |
| `-PS<ports>` | TCP SYNパケットでのホスト検出（例: `-PS22,80,443`） |
| `-PA<ports>` | TCP ACKパケットでのホスト検出 |
| `-PU<ports>` | UDPパケットでのホスト検出 |
| `-PE` | ICMP Echo Requestでのホスト検出 |
| `-PP` | ICMP Timestamp Requestでのホスト検出 |
| `-PY` | SCTPプロトコルでのホスト検出 |
| `-PO` | IPプロトコルpingでのホスト検出 |
| `-n` | DNS逆引き解決をしない（高速化） |
| `-R` | 常にDNS逆引きを行う |

```bash
# ネットワーク全体で生きているホストのみ検出
nmap -sn 192.168.1.0/24

# ファイアウォール環境でping無効時の強制スキャン
nmap -Pn -p 80,443 192.168.1.10
```

---

## 2. ポートスキャンタイプ

| オプション | 説明 |
|---|---|
| `-sS` | TCP SYNスキャン（ステルス性が高く最も一般的、要root権限） |
| `-sT` | TCP Connectスキャン（フルコネクト、非root可） |
| `-sU` | UDPスキャン（DNS, SNMPなどのサービス検出に必須） |
| `-sA` | TCP ACKスキャン（ファイアウォールルール確認用） |
| `-sW` | TCP Windowスキャン |
| `-sM` | TCP Maimonスキャン |
| `-sN` / `-sF` / `-sX` | Null / FIN / Xmasスキャン（IDS/FW回避目的） |

---

## 3. ポート指定オプション

| オプション | 説明 |
|---|---|
| `-p <port>` | 特定ポート指定（例: `-p 80,443`） |
| `-p-` | 全65535ポートをスキャン |
| `-p 1-1000` | 範囲指定 |
| `-F` | よく使われる100ポートのみ（高速） |
| `--top-ports <n>` | 使用頻度の高い上位n個のポート |

---

## 4. サービス・バージョン検出

| オプション | 説明 |
|---|---|
| `-sV` | サービスとバージョンを検出 |
| `--version-intensity <0-9>` | 検出精度（高いほど詳細だが時間がかかる） |
| `-O` | OS検出（フィンガープリンティング） |
| `-A` | OS検出+バージョン検出+スクリプトスキャン+traceroute（総合スキャン） |

---

## 5. Nmap Scripting Engine (NSE)

| オプション | 説明 |
|---|---|
| `-sC` | デフォルトの安全なスクリプトを実行（`--script=default`と同義） |
| `--script=<script name>` | 特定スクリプト指定（例: `--script=vuln`） |
| `--script=vuln` | 既知の脆弱性を検出するスクリプト群 |
| `--script=auth` | 認証関連の脆弱性検出 |
| `--script-args=<args>` | スクリプトへの引数指定 |

```bash
# 脆弱性スキャンの代表例
nmap -sV --script=vuln 192.168.1.10
```

---

## 6. タイミング・パフォーマンス制御

| オプション | 説明 |
|---|---|
| `-T0`〜`-T5` | スキャン速度（0=最も遅く検出されにくい、5=最速） |
| `--min-rate <n>` | 最小パケット送信レート |
| `--max-retries <n>` | 再送信回数の上限 |

ペンテストではIDS回避のため `-T2` や `-T3` を使うケースもありますが、業務時間の都合で `-T4` がよく使われます。

---

## 7. 出力オプション

| オプション | 説明 |
|---|---|
| `-oN <file>` | 通常形式で保存 |
| `-oX <file>` | XML形式で保存（他ツール連携用） |
| `-oG <file>` | grep可能形式 |
| `-oA <basename>` | 上記全形式で同時出力 |

---

## 8. ファイアウォール／IDS回避関連オプション

| オプション | 説明 |
|---|---|
| `-f` | パケットをフラグメント化 |
| `-D <decoy1,decoy2,...>` | おとりIPを使い送信元を偽装 |
| `-S <IP>` | 送信元IPを偽装 |
| `--source-port <port>` | 送信元ポートを指定 |
| `--data-length <n>` | パケットにランダムデータを付加 |

※これらはIDS/IPSでの検出回避技術のため、**必ず許可された範囲・契約内のペンテストでのみ**使用してください。

---

## 実践例（典型的なペンテストの流れ）

```bash
# 1. ホスト検出
nmap -sn 192.168.1.0/24

# 2. 開いているポートの全体スキャン
nmap -Pn -p- -T4 192.168.1.10

# 3. サービス・バージョン・OS検出
nmap -sV -O -p 22,80,443 192.168.1.10

# 4. 脆弱性スクリプトスキャン
nmap -sV --script=vuln -p 22,80,443 192.168.1.10

# 5. 結果を保存
nmap -A -oA scan_result 192.168.1.10
```

---

## 注意点

- nmapによるスキャンは**対象の所有者から明示的な許可**を得た環境（自社ネットワーク、正式契約のあるペンテスト業務、CTF/検証用ラボなど）でのみ実施してください。無許可のスキャンは不正アクセス禁止法等に抵触する可能性があります。
- `-sS`や`-O`などはroot権限（Linux）または管理者権限（Windows）が必要です。
- 本番環境への`-T4`/`-T5`や`-p-`スキャンはサービスに負荷をかける場合があるため、事前に影響範囲を確認してください。

nmapのオプション命名規則は基本的に一貫していて、大文字の後に続く文字がカテゴリ（グループ）を、小文字が具体的な手法を表します。

始めに押さえておくべき「頻出コアオプション」を優先度別にまとめます。

## 最優先

| オプション | 用途 | 覚え方 |
|---|---|---|
| `-sS` | SYNスキャン(標準) | 最も使うスキャン方式 |
| `-sV` | バージョン検出 | 何のサービスが動いているか |
| `-O` | OS検出 | 対象OSの特定 |
| `-p-` / `-p <port>` | ポート範囲指定 | 全ポート or 特定ポート |
| `-A` | 上記+スクリプト+tracerouteを一括 | 総合偵察の決定版 |
| `-Pn` | ホスト検出スキップ | FWでping拒否時に必須、実務で頻出 |
| `-sn` | ポートスキャンせずホスト検出のみ | サブネット全体の生存確認 |
| `-sC` | デフォルトNSEスクリプト | 手軽な情報収集 |
| `-oN`/`-oX`/`-oA` | 結果保存 | レポート作成に必須 |
| `-T4` | 速度調整 | 実務で最も使う速度設定 |

この10個で実務の8割はカバーできます。

## 次に優先

| オプション | 用途 |
|---|---|
| `-sU` | UDPスキャン(SNMP/DNS調査などで重要、忘れがち) |
| `--script=vuln` | 脆弱性スキャン |
| `-F` | 高速スキャン(上位100ポート) |
| `--top-ports <n>` | 頻出ポートのみ |
| `-v` / `-vv` | 詳細出力(進捗確認) |
| `--min-rate` | 大規模スキャンの高速化 |

## 覚える優先順位が低い(存在を知っておく程度でOK)

- `-D`(デコイ)、`-f`(フラグメント)、`-S`(送信元偽装)などのIDS回避系
- `-sN`/`-sF`/`-sX`(特殊スキャン、実務での使用頻度は低い)
- `-sW`/`-sM`/`-sA`(ACK/Window/Maimon、FWルール調査など専門的用途)

---

## 実務で最初に叩くコマンド(これを覚えておけば大抵対応可能)

```bash
# ホスト発見
nmap -sn 192.168.1.0/24

# 初期偵察(全ポート高速)
nmap -Pn -p- --min-rate=1000 -T4 <target>

# 詳細スキャン(見つかったポートに対して)
nmap -Pn -sV -sC -O -p <found_ports> <target>
```

このパターンをまず体に染み込ませるのがおすすめです。オプションを個別に暗記するより、「ホスト発見→全ポートスキャン→詳細スキャン」という**流れ**を覚えると自然とオプションも定着します。
