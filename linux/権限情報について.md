## 権限情報について
Linux/Unixの権限(パーミッション)情報の読み方を解説します。

## `ls -l` で表示される権限情報

```bash
$ ls -l
-rwxr-xr-- 1 user group 4096 Jul 15 10:00 sample.txt
```

この先頭の `-rwxr-xr--` の部分が権限情報です。

## 10文字の構成

```
-  rwx  r-x  r--
①   ②    ③    ④
```

| 位置 | 意味 |
|---|---|
| ① | ファイルの種類 |
| ② | 所有者(owner/user)の権限 |
| ③ | グループ(group)の権限 |
| ④ | その他(other)の権限 |

### ① ファイルの種類

| 文字 | 意味 |
|---|---|
| `-` | 通常のファイル |
| `d` | ディレクトリ |
| `l` | シンボリックリンク |
| `c` | キャラクタデバイス |
| `b` | ブロックデバイス |

### ②③④ 権限(3文字ずつ)

それぞれ3文字が `r`(read)、`w`(write)、`x`(execute)の有無を表します。

| 文字 | 意味 | ファイルの場合 | ディレクトリの場合 |
|---|---|---|---|
| `r` | 読み取り | 内容を見られる | 中身の一覧を見られる |
| `w` | 書き込み | 内容を編集できる | 中にファイルを作成・削除できる |
| `x` | 実行 | プログラムとして実行できる | ディレクトリに入れる(`cd`できる) |
| `-` | 権限なし | 該当操作不可 | 該当操作不可 |

### 具体例

```
-rwxr-xr--
```

- 所有者(`rwx`):読み・書き・実行すべて可能
- グループ(`r-x`):読み・実行は可能、書き込み不可
- その他(`r--`):読み取りのみ可能

## 数値(8進数)表記

`chmod` でよく使う数値表記は、`r=4`、`w=2`、`x=1` の合計で表します。

| 記号 | 数値 | 意味 |
|---|---|---|
| `rwx` | 4+2+1=7 | 読み書き実行すべて可 |
| `rw-` | 4+2=6 | 読み書き可、実行不可 |
| `r-x` | 4+1=5 | 読み実行可、書き込み不可 |
| `r--` | 4 | 読み取りのみ |
| `-wx` | 2+1=3 | 書き実行可、読み取り不可 |
| `---` | 0 | 権限なし |

### 例:`chmod 754 file`

```
7 → rwx (所有者)
5 → r-x (グループ)
4 → r-- (その他)
```

つまり `-rwxr-xr--` と同じ意味になります。

## よく使う組み合わせ

| 数値 | 記号 | 用途 |
|---|---|---|
| `755` | `rwxr-xr-x` | 実行スクリプトやディレクトリの標準的な権限 |
| `644` | `rw-r--r--` | 一般的なファイル(所有者のみ編集可) |
| `700` | `rwx------` | 所有者のみアクセス可能(秘密鍵など) |
| `600` | `rw-------` | 所有者のみ読み書き可能 |
| `777` | `rwxrwxrwx` | 全員が全権限(セキュリティ上は非推奨) |

## 特殊な権限(応用)

`ls -l` の実行権限の位置に `s` や `t` が表示される場合があります。

```
-rwsr-xr-x   # SUID(所有者権限で実行)
-rwxr-sr-x   # SGID(グループ権限で実行)
drwxrwxrwt   # スティッキービット(削除制限、/tmpなどでよく見る)
```

| 記号 | 名称 | 意味 |
|---|---|---|
| `s`(所有者のxの位置) | SUID | 実行時に所有者の権限で動作する |
| `s`(グループのxの位置) | SGID | 実行時にグループの権限で動作する/ディレクトリ内で新規作成されるファイルのグループを継承 |
| `t`(その他のxの位置) | Sticky Bit | ディレクトリ内で他人のファイルを削除できないようにする(`/tmp`など) |

数値表記では4桁目として指定します(例:`chmod 4755 file` でSUID付与)。

## 所有者・グループの確認

```bash
$ ls -l sample.txt
-rwxr-xr-- 1 user group 4096 Jul 15 10:00 sample.txt
              ^^^^ ^^^^^
              所有者 グループ
```

`user` がファイルの所有者、`group` が所属グループを示しています。この所有者・グループが誰であるかによって、②③④のどの権限が適用されるかが決まります。

## まとめ図

```
-rwxr-xr--
│└┬┘└┬┘└┬┘
│ │  │  └─ その他(other)の権限
│ │  └──── グループ(group)の権限
│ └─────── 所有者(owner)の権限
└───────── ファイル種別
```

この読み方が分かれば、`chmod` で権限を変更する際にも、どの数値・記号を指定すればよいか判断しやすくなります。
